# Default Preferences

Starting defaults for new users. These get overridden as the journal accumulates real preferences.

## Briefing Structure

- **Sections:** calendar, tasks (if available), news (only after topic preferences are established)
- **Format:** bullets
- **Length:** under 20 lines on a normal day
- **Lead with:** what changed since yesterday

## Tone

- Concise, direct
- No greetings, no sign-offs
- State facts, flag problems, suggest actions
- Use plain language - no corporate jargon

## Calendar

- Show next 24 hours
- Flag meetings needing prep
- Flag scheduling conflicts
- Note focus time gaps (90+ min blocks)

## Tasks

- Top 3 only
- Include days overdue for late items
- Don't repeat tasks mentioned yesterday unless status changed

## News

- Disabled until the user shows interest in a topic
- When enabled: 2-3 items max, one sentence each
- Only topics with 3+ days of engagement in the journal
